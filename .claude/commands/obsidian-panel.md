# Obsidian Panel

For decisions where you're unsure or want to stress-test your thinking.

## Step 1 — Load context
Read `SOUL.md` and `CRITICAL_FACTS.md`. Read the relevant project or area notes.

## Step 2 — Define the decision
Ask the user to state the decision clearly in one sentence if not already given.

## Step 3 — Convene the panel
Run 3-4 distinct perspectives. Each argues from a different stance:

**Perspectives to use (pick 3-4 based on the decision):**
- **The Operator** — what's most efficient, lowest risk, fastest path to revenue?
- **The Contrarian** — what's wrong with the obvious answer? what are you missing?
- **The Customer** — what would the end user or client actually experience?
- **The Long Game** — what does this look like in 12 months if done wrong? done right?
- **The Frugalist** — what's the cheapest version that still works?

Format for each:
```
### [Perspective Name]
[3-5 sentences arguing their position, using vault context where relevant]
**Verdict:** [specific recommendation]
```

## Step 4 — Synthesize
After all perspectives: state the tension, then give a final recommendation that resolves it. Note what was sacrificed.

## Step 5 — Offer to save
Ask if the decision should be logged via `/obsidian-save`.
